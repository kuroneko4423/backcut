# 07. 用語集

> Claude Code がドメイン特有の用語を誤解しないための辞書。

---

## アプリ機能関連

| 用語 | 意味 |
|---|---|
| **背景透過** | 画像から背景部分を識別し、アルファチャンネルを0にして除去する処理 |
| **レタッチ** | 自動透過の結果を手動で補正する作業（戻す・消す） |
| **「戻す」モード** | 自動透過で消えすぎた領域を、オリジナル画像から復元するブラシモード |
| **「消す」モード** | 自動透過で消し残った領域を、ブラシで透明化するモード |
| **自動トリミング** | 透明領域を検出して、画像周囲の余白を自動カットする処理 |
| **手動トリミング** | ユーザーがマウスドラッグで矩形範囲を指定して切り出す処理 |
| **エッジ調整** | 切り抜き境界（エッジ）の硬さ・滑らかさを調整する処理 |
| **ドロップシャドウ** | 画像の下に影を合成し、立体感や接地感を出す処理 |
| **立ち絵プリセット** | キャラクターの足元接地影を想定した影パラメータ |
| **アイテムプリセット** | 小物・商品画像用の控えめな影パラメータ |
| **扁平度（flat）** | 影をY方向に潰す度合い。小さいほど地面接地に近い |
| **スプレッド（spread）** | 影の横方向スケール |

## 技術用語

| 用語 | 意味 |
|---|---|
| **Canvas API** | ブラウザ標準の2D描画API。`<canvas>` 要素に対する `getContext('2d')` 経由で使用 |
| **ImageData** | Canvas上のピクセルデータ（RGBA配列）を直接操作するためのオブジェクト |
| **アルファチャンネル** | 各ピクセルの透明度。0（完全透明）〜255（完全不透明） |
| **ONNX** | Open Neural Network Exchange。機械学習モデルの共通フォーマット |
| **ONNX Runtime Web** | ブラウザ上でONNXモデルを実行するランタイム |
| **WebAssembly (Wasm)** | ブラウザ上でネイティブに近い速度で実行できるバイナリ形式 |
| **WebGPU** | ブラウザ上でGPUを直接利用する新しいAPI（高速推論に活用） |
| **OffscreenCanvas** | DOMに紐づかないCanvas。Web Workerでも使える |
| **IndexedDB** | ブラウザ内の大容量データ保存用DB。モデルファイルのキャッシュに使う |
| **CSP** | Content Security Policy。読み込めるリソース・実行できるスクリプトを制限するHTTPヘッダ |
| **COOP / COEP** | Cross-Origin-Opener-Policy / Cross-Origin-Embedder-Policy。SharedArrayBuffer 等の利用に必要 |

## 注意したい用語の混同

- **「背景透過」≠「背景削除」**：本アプリでは「透過」を使う（背景部分のアルファを0にするだけで、ピクセルは残る場合もある）
- **「PNG」≠「JPEG」**：透過情報を維持するならPNGまたはWebPが必須。JPEGでは透過不可
- **「Canvas」と「canvas要素」**：単に「Canvas」と書く場合は描画コンテキストを指す。HTML要素は「`<canvas>` 要素」と明示する
