## 1. プロジェクト概要

| 項目 | 内容 |
|---|---|
| アプリ名 | BackCut |
| 種別 | ブラウザ完結型 背景透過＋画像編集アプリ |

## 4. 非機能要件

### 性能
- 1024×1024 画像で背景透過処理は **10秒以内** が目標
- WebGPU対応ブラウザでは推論を高速化、非対応時は WebAssembly フォールバック

### セキュリティ（必ず守る）
1. **画像データの外部送信禁止**（コア原則）
2. **CSP ヘッダ設定**（`next.config.js` で設定）
3. **依存ライブラリの脆弱性チェック**：`npm audit` を定期実行
4. **3段階セキュリティチェック**：詳細は `06_deploy-and-security.md`

### プライバシー
- Cookie・トラッキングは原則使わない
- localStorage には**アプリ設定のみ**保存。画像本体は保存しない
- IndexedDB には**モデルファイルのキャッシュのみ**保存

### 互換性
- **対応**: Chrome / Edge / Firefox 最新版、Safari 16+
- **推奨**: メモリ8GB以上、デスクトップブラウザ
- **制限あり**: iOS Safari（WebAssemblyのメモリ制約により大画像処理が遅い場合あり）

---

## 5. 主要な制約

- 背景透過モデルファイル（数十〜数百MB）の初回ダウンロードが発生する
- 巨大画像（長辺2048px超）は処理前に内部リサイズする
- 透過情報の維持には PNG または WebP 出力が必須（JPEGは不可）

---

## 6. スコープ外（やらないこと）

- バックエンドAPI / サーバー処理
- ユーザー認証 / ログイン
- 画像のクラウド保存
- 多言語切替（日本語のみで開始、将来検討）
- AIによるエッジ自動最適化
- 本格的なレイヤー機能

---

## 7. 完成の定義（Definition of Done）

- 機能 F-01 〜 F-11 がVercel本番環境で動作する
- DevTools Network タブで画像データの外部送信が発生しないことを確認できる
- Chrome / Firefox / Safari で動作確認済み
- `npm run lint` および `npm run build` がエラーなく通る
- README が整備されている
